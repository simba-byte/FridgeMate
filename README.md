# FridgeMate：智能冰箱食材管理与菜谱推荐系统

## 一、项目背景

FridgeMate 是一个面向家庭冰箱管理场景的 Android 原生课程设计项目。日常生活中，用户经常会忘记冰箱中已有的食材、无法及时处理临期食品，也很难根据现有库存快速决定今天吃什么。本项目通过本地数据持久化记录食材信息，并结合轻量级菜谱推荐算法，帮助用户减少食物浪费、提升冰箱管理效率。

## 二、功能介绍

系统包含五个主要页面：

1. 首页 Dashboard：展示食材总数、即将过期数量、已过期数量、优先食用提醒和今日推荐菜谱，并提供快速添加入口。
2. 食材管理：支持食材新增、编辑、删除、标记已使用、分类筛选、关键词搜索，并根据过期日期自动判断状态。
3. 菜谱推荐：根据当前未过期、未使用的食材与本地预置菜谱材料进行匹配，按匹配度展示推荐结果。
4. 购物清单：支持添加购物项、删除购物项、标记已购买/未购买，已购买项会以置灰和删除线展示。
5. 统计分析：展示食材总数、状态数量、分类统计和食材利用率，并使用 ProgressBar 展示利用率。

## 三、技术路线

- 开发语言：Java
- 界面技术：XML Layout、Activity、Fragment、RecyclerView、Adapter
- 数据持久化：Room 数据库
- UI 组件：Material Components、BottomNavigationView、MaterialCardView、FloatingActionButton
- 数据架构：Entity + DAO + AppDatabase + Repository
- 运行方式：Android Studio 打开并同步 Gradle 后运行

## 四、系统结构

主要包结构如下：

```text
com.example.fridgemate
├── activity
├── adapter
├── database
├── entity
├── fragment
├── repository
└── utils
```

其中 Activity 负责页面入口和表单流程，Fragment 负责底部导航中的五个主页面，Adapter 负责列表展示，Repository 封装后台线程中的数据库操作，utils 提供日期、状态判断和菜谱匹配算法。

## 五、数据库设计

项目使用 Room 创建本地数据库 `fridgemate.db`，包含以下表：

1. `food_items`：保存食材名称、分类、数量、单位、存放位置、购买日期、过期日期、状态、备注、创建时间。
2. `recipes`：保存本地预置菜谱名称、所需材料、制作步骤、难度和预计时间。
3. `shopping_items`：保存购物项名称、分类、数量、单位、是否已购买和创建时间。

首次启动时，系统会自动写入 10 条默认菜谱，并在食材表为空时写入几条示例食材，方便打开 App 后直接看到 Dashboard、菜谱匹配和统计效果。

## 六、核心模块说明

食材状态由 `StatusUtils` 根据 `expireDate` 自动生成：

- 当前日期超过过期日期：已过期
- 距离过期日期小于等于 3 天：即将过期
- 其他情况：正常
- 用户手动标记后：已使用

所有数据库操作均通过 DAO 执行，并由 Repository 调用 `ExecutorService` 放入后台线程，避免阻塞主线程。

## 七、菜谱推荐算法说明

本系统采用基于食材关键词匹配的轻量级菜谱推荐算法。

算法流程：

1. 读取当前数据库中未过期、未使用的食材名称。
2. 将菜谱 `materials` 字段按顿号、逗号拆分为材料列表。
3. 逐个判断菜谱材料是否能与已有食材名称匹配。
4. 计算匹配度：`匹配数量 / 菜谱材料总数 * 100%`。
5. 根据匹配度从高到低排序。
6. 匹配度 100% 时显示“可直接制作”；部分匹配时显示缺少的食材；完全不匹配时排在后面。

该算法不依赖网络和真实 AI 服务，适合课程项目展示，同时具备清晰的可解释性和较低实现复杂度。

## 八、项目运行方法

1. 使用 Android Studio 打开 `D:\Desktop\terminal`。
2. 等待 Gradle Sync 完成。
3. 选择模拟器或真机设备。
4. 点击 Run 运行 App。

如本机没有缓存依赖，首次 Gradle Sync 需要 Android Studio 下载 Android Gradle Plugin、AndroidX、Room 和 Material Components 依赖。

## 九、验收点对应

- Room 数据库：已实现 `FoodItem`、`Recipe`、`ShoppingItem` 及对应 DAO。
- 首页统计：展示总数、临期、过期、优先食用和推荐菜谱。
- 食材 CRUD：支持新增、编辑、删除、已使用标记、搜索和筛选。
- 菜谱推荐：根据库存食材动态计算匹配度。
- 购物清单：支持添加、删除、勾选已购买。
- 统计分析：展示状态统计、分类统计和利用率。
- 本地持久化：数据关闭 App 后仍保存在 Room 数据库中。
